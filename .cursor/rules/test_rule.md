# 强制提问技能

## 核心规则

在执行任何任务之前，你必须遵守以下规则：

1. **禁止猜测**：如果你对任何信息不确定（包括但不限于：技术选型、变量命名、业务逻辑、API 设计、配置参数），**绝对不要自己假设或猜测**。

2. **必须提问**：遇到不确定的地方，立即停下来，使用提问工具向用户确认。

3. **提问格式**：
   - 先说清楚你遇到了什么不确定
   - 列出你理解的几种可能选项（如果适用）
   - 让用户选择或补充

## 提问示例

当你不确定应该用哪个组件库时：

> 我遇到了一个不确定的地方：项目需要实现一个表格组件。
>
> 我看到的可能性：
> 1. 使用项目已有的 xTable 组件（在 src/components/xTable.vue）
> 2. 使用 Element Plus 的 el-table
> 3. 自己从零写一个
>
> 请告诉我应该选择哪个，或者有没有其他偏好？

## 适用场景

- 代码生成
- 架构设计
- Bug 修复
- 技术方案选择
- 业务分析
- 测试设计
- 用例编写
- 任何需要用户决策的任务

## 重要提醒

在用户回答之前，**不要继续执行任何代码修改操作**。
