## BUG-1: 可将父部门的上级设为其子部门（循环引用） 【待确认】

- **关联用例**: TC-G7 (`test_set_parent_to_descendant_007`)
- **复现步骤**:
  1. 创建部门 A（deptId=631）
  2. 在 A 下创建子部门 B（deptId=632，parentId=631）
  3. 修改 A 的 parentId 为 632（将父部门的上级设为其子部门）
- **实际结果**: 系统返回 `{"msg": "操作成功", "code": 200}`，修改成功
- **预期结果**: 系统应返回 500 拒绝，不允许将部门的上级设为其子孙部门（会造成循环引用，破坏树结构）
- **影响**: 部门树结构被破坏，ancestors 字段异常，可能导致查询死循环、级联操作失败、权限数据范围计算错误
- **严重级别**: 高
- **BUG 类型**: 业务规则
- **脚本断言**: `check=[["$.code", "eq", 200]]`（脚本未预期此为错误行为，ISS-2 问题已记录）
- **确认证据**: 日志 `test_dept_hierarchy_advanced_20260307_231250.log`：`mod_dept` 请求 `{"deptId": 631, "parentId": 632}` → 响应 `{"msg": "操作成功", "code": 200}`。注：系统已能拦截"上级部门不能是自己"（TC-B4），但未拦截设为**子孙**节点的间接循环。

## BUG-2: 部门名称前后空格未做 trim 处理 【待确认】

- **关联用例**: TC-J13 (`test_name_with_spaces_013`)
- **复现步骤**: 创建部门，deptName 传入 `" sp_j13_96369_897 "`（前后含空格）
- **实际结果**: 系统返回 200，含空格的名称原样入库存储为 `" sp_j13_96369_897 "`
- **预期结果**: 系统应自动 trim 前后空格后入库，即存储为 `"sp_j13_96369_897"`
- **影响**: 含空格的名称会影响唯一性校验（"部门A" 和 " 部门A " 被视为不同部门）、搜索结果不准确、前端展示异常
- **严重级别**: 中
- **BUG 类型**: 数据完整性
- **脚本断言**: 脚本仅断言 `$.code == 200`（ISS-2 问题，未验证存储值）
- **确认证据**: 日志 `test_dept_field_validation_20260307_231249.log`：请求 `"deptName": " sp_j13_96369_897 "` → 响应 code=200 → 详情接口返回 `"deptName": " sp_j13_96369_897 "`（空格原样保留）
